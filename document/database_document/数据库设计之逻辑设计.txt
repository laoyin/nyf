数据库设计之逻辑设计

1. 逻辑设计任务
1.1 实体型转化为关系模式
1.2 实体间的联系转换为关系模式
1.3 确定关系模式的属性和码

2. 转换规则
2.1 一个实体型转换为一个关系模式
2.2 一个1:1联系可以和任意一端的关系模式合并
2.3 一个1:n联系与n端对应的关系模式合并或单独转换为一个关系模式
2.4 一个m:n转换为一个关系模式 
关系的属性 : 与该联系相连的各实体的码以及联系本身的属性
关系的码 : 各实体的码	
2.5 三个或三个以上的实体间的多元联系转换为一个关系
关系的属性 : 与该联系相连的各实体的码以及联系本身的属性
关系的码 : 各实体的码	

3.  优化数据模型 R(U,D,DOM,F) ==> R(U,F)
3.1 一个好的关系模式的特征 
尽可能少的数据冗余
没有插入异常
没有删除异常
没有更新异常

3.2 关系模式的规范化
将低一级范式的关系模式,通过模式分解转换为高一级范式的关系模式集合的过程叫关系模式的规范化
1NF : 数据项不可分
2NF : 所有非主属性都完全依赖于候选码(主键)
3NF : 满足2ＮＦ且任何非主属性都不传递依赖于候选码
BCNF : 满足3NF且消除主属性对候选码的部分函数依赖
